# Skill: documentation-writer

## Goal

Produce documentation that matches **how the repo actually works**.

## Method

1. Verify claims against files/commands in the repo.
2. Prefer short examples that can be copy-pasted.
3. Link paths with backticks: `agent/cli.py`.

## Guardrails

- Avoid marketing language; prefer operational clarity.
- Call out security-relevant steps explicitly.
